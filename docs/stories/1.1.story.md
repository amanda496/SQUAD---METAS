# Story 1.1: Implementar Regras de Cálculo de Score — Squad Fechamento de Metas

## Status

Done

## Story

**As a** gestora da Stark (Amanda),
**I want** que o sistema de fechamento de metas calcule o score ponderado com as regras corretas de threshold e binário para cada KPI,
**so that** a remuneração variável dos gestores de tráfego reflita fielmente as regras de negócio vigentes (ROI/CPS zerados se <70%, Contato/SR zerados se <100%).

## Acceptance Criteria

1. ROI ≥ 1 (peso 15%): só contribui se o percentual de clientes que bateram a meta for ≥ 70%; caso contrário, contribuição = 0%.
2. CPS < R$2,59 (peso 20%): só contribui se o percentual de clientes que bateram a meta for ≥ 70%; caso contrário, contribuição = 0%.
3. Contato com Cliente (peso 15%): só contribui se 100% dos clientes foram contatados; qualquer valor < 100% → contribuição = 0%.
4. Status Report (peso 15%): só contribui se a task foi 100% concluída; qualquer valor < 100% → contribuição = 0%.
5. A fórmula do Score Ponderado na planilha Google Sheets (aba Abril, coluna J) deve refletir todas as regras acima.
6. Os scores recalculados do mês de Abril devem estar atualizados na aba "1º Semestre", coluna I (Abril).
7. Os arquivos `kpis.yaml` e `fechar-mes.md` do squad devem documentar as regras de threshold e binário.
8. O dado de Contato com Cliente do Gustavo Radler (H5 na aba Abril) deve estar corrigido para 100%.

### Regras de Negócio Implementadas

**Fórmula do Score Ponderado (planilha Abril, coluna J):**
```
=SEERRO(
  C3*0.125                    ← NPS (12.5%)
  + D3*0.125                  ← Churn (12.5%)
  + IF(E3>=0.7,E3,0)*0.15     ← ROI ≥1 (15%) — threshold 70%
  + IF(F3>=0.7,F3,0)*0.2      ← CPS <R$2,59 (20%) — threshold 70%
  + G3*0.1                    ← Google CPL (10%) — proporcional
  + IF(H3=1,1,0)*0.15         ← Contato (15%) — binário 100%
  + IF(I3=1,1,0)*0.15         ← Status Report (15%) — binário 100%
,"")
```
> Nota: Digitar com `IF()` — Google Sheets converte automaticamente para `SE()` ao exibir.

**Atenção para entrada de dados na aba Abril (células % formatadas):**
- Células A-I armazenam decimais (0 a 1): `1 = 100%`, `0.7 = 70%`
- Para entrar 100% em célula formatada como %: digitar `100` (armazena 1 = 100%)
- Não digitar `1` (seria 1%) nem `100%` (pode causar erro de parsing)

**Atenção para a aba 1º Semestre (coluna I — Abril):**
- Células armazenam o score como número inteiro/decimal sem % (ex: 52.5 = 52.5%)
- Não entrar como decimal (0.525 resultaria em 0.525%, não 52.5%)
